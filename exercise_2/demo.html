Section 2: (Margin and Padding)
This element is the same as the red element in Section 1 but has a margin of 5px. Notice how the margin "pushes" the red box away from the black border. The grey background surrounding the red box is created by the use of margin. If you inspect this element, you can see the layout engine providing the margin. This is done like this:
.margin-demo{ margin: 5px; }
This element is the same as the orange element in section 1 except it has a padding of 5px. This element has NO margin, unlike the element above, so the borders of the orange box are back to "hugging" the containers. This is done like this:
.padding-demo{ padding: 5px; }
This element has both a margin and padding of 5px. Notice how much better this looks than the elements that are slammed up against the borders.
The TL;DR for margin in padding on block level elements is this:
Margin "pushes" this element away from others by the specified amount
Padding "pushes" the content within an element away from the edges of the element
It is also worth noting that margins and padding can be applied individually to each side of the element. This element has 10px of margin on the left side and 50px of padding on the top. This is done like this:
.padding-demo{ padding-top: 50px; margin-left: 10px; }
The same technique can be used for the bottom and right of elements as well. Note that the grey space above this element is created by the margin of the box ABOVE it and not this element itself

Section 3 (flexbox)
Inline-block and inline still had some shortcomings when it came to laying out HTML elements. A new method of laying out pages was created to try to solve this called flexbox. Flexbox allows for flexible and easily created column and row base layouts. We mark an element as a flex layout element by giving it the property display:flex
This element is marked as having a flex layout as well as a margin and padding property, flex elements are laid out similarly to inline elements

Flex box also provides us with the ability to set how the content inside of a container is justifed. This is done using the justify-content and align-items properties.

Justify content determines how items are laid out along the main axis of the container. If the container is marked as flex-direction: row, then justify-content will determine the position of child elements from left to right. If the container is marked as flex-direction: column, then justify-content will determine the position of the child from top to bottom.

align-items is the same but for the cross-axis. If the container is marked as flex-direction: row, then align-items will determine the position of child elements from top to bottom. If the container is marked asflex-direction: column, then align-items determines the position of the child from left to right.
